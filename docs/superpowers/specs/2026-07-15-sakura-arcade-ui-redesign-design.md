# 樱花街机学院 · UI重设计规范文档

## 1. 目标与评分价值

### 1.1 设计目标

本规范定义"樱花街机学院"全站UI重设计方案，旨在将现有二次元浓度测试应用改造为高品质日式游戏风格界面，核心目标：

- **视觉风格转型**：从现有柔和粉色系转向深靛粗描边、硬边阴影、糖果色面板的街机游戏UI风格
- **游戏化界面**：引入卡牌、HUD、贴纸徽章、角色卡等游戏元素，提升用户沉浸感
- **功能完整性**：保持所有现有功能逻辑不变，仅进行视觉重构
- **响应式适配**：桌面端游戏主界面、平板两列压缩、移动端单列布局

### 1.2 评分标准适配

| 评分维度 | 适配策略 | 实现方式 |
|---------|---------|---------|
| 创新性 | 街机游戏风格融合二次元主题 | 深靛描边+硬边阴影+糖果色面板+贴纸徽章 |
| 实用性 | 清晰的数据展示与操作路径 | HUD状态条+任务面板+答案技能卡+通关证书 |
| 完成度 | 全页面覆盖，功能与数据边界明确 | 首页/模式选择/答题/结果/弹窗全覆盖 |
| 设计 | 高品质游戏UI规范 | 统一组件样式、视觉令牌、动画系统 |

### 1.3 设计参考

参考游戏UI风格元素：
- 深靛粗描边：2-4px粗边框，#0B1220深靛色
- 硬边阴影：无模糊边缘，清晰的层级阴影
- 糖果色面板：高饱和度纯色面板，#FF8FA3樱花粉、#7ED4F5晴空蓝、#B388EB薰衣草紫、#81E8D0薄荷绿、#FFD966蜂蜜金
- 贴纸徽章：不规则形状、倾斜角度、胶带固定效果
- 卡牌布局：圆角矩形、层级堆叠、稀有度标识
- HUD界面：状态栏、计时器、分数显示
- 角色卡：头像+属性+技能展示

## 4. 组件清单

### 4.1 核心组件

| 组件名称 | 类型 | 状态 | 视觉特征 |
|---------|------|------|---------|
| **HUD状态栏** | 固定顶部 | 常驻 | 深靛背景、糖果色数据块、粗描边 |
| **模式卡牌** | 卡片 | 正常/选中/悬停 | 不规则圆角、胶带装饰、稀有度星星 |
| **题目卡牌** | 卡片 | 进入动画 | 顶部徽章、粗描边、硬边阴影 |
| **答案技能卡** | 按钮 | 正常/悬停/正确/错误 | 彩色按键标识、硬边阴影 |
| **任务面板** | 侧边面板 | 展开/收起 | 糖果色标签、进度条 |
| **通关证书** | 主展示 | 结果页 | 丝带装饰、印章效果、粗描边 |
| **段位徽章** | 徽章 | 不同段位 | 圆形/星形、糖果色、贴纸风格 |
| **角色卡组** | 卡片组 | 结果页 | 头像+属性+匹配度 |
| **游戏弹窗** | 模态框 | 打开/关闭 | 统一面板样式、粗描边 |

### 4.2 组件样式规范

**HUD状态栏**：
- 高度：60px
- 背景：#0B1220
- 内容布局：左侧返回按钮、中间标题、右侧数据块（分数/时间/模式）
- 数据块样式：糖果色背景（#FFD966计时、#81E8D0分数、#7ED4F5模式）、白色粗体文字、胶囊圆角

**模式卡牌**：
- 尺寸：等宽网格布局
- 背景：#F7F5EF
- 边框：3px #0B1220粗描边
- 圆角：20px 8px 24px 12px
- 装饰元素：顶部胶带（半透明条纹）、左上角稀有度星星、右上角选中印章

**题目卡牌**：
- 背景：#FFFFFF
- 边框：3px #0B1220粗描边
- 圆角：16px
- 顶部徽章：渐变背景（题型对应颜色）、白色文字、倾斜角度
- 阴影：硬边阴影+浮起效果

**答案技能卡**：
- 背景：#F7F5EF
- 边框：2px #0B1220描边
- 按键标识：圆形彩色背景（A=晴空蓝、B=薰衣草紫、C=薄荷绿、D=蜂蜜金）、白色字母
- 选中状态：边框变色、阴影加深
- 正确状态：薄荷绿边框+背景
- 错误状态：阻塞红边框+背景+抖动动画

**通关证书**：
- 背景：线性渐变（#FFFEFA → #FFF9F0）
- 边框：3px #0B1220粗描边 + 内部虚线边框
- 顶部丝带：三角形剪裁、樱花粉渐变
- 印章：圆形、樱花粉、倾斜角度

## 6. 数据绑定与不变式

### 6.1 核心数据绑定

| UI元素 | 数据源 | 绑定规则 |
|-------|-------|---------|
| 题库数量 | CONFIG.questions.length | 动态计算，显示真实数值 |
| 成就数量 | Storage.get().achievements.unlocked.length | 实时更新 |
| 计时器 | Quiz.timer | 游戏内计时，格式MM:SS |
| 进度条 | (currentIndex + 1) / totalQuestions | 百分比计算 |
| 分数 | Scoring.getScore() | 实时计算，含基础分+竞技分 |
| 段位 | Rank.calculate() | 根据分数映射段位 |
| 知识掌握度 | Scoring.getCategoryScores() | 基础分加权准确率 |
| 竞技表现 | Scoring.getCompetitionScore() | 含速度分+连击分 |
| 分类画像 | Scoring.getCategoryScores() | 按分类统计，样本规则适用 |
| 角色匹配 | CharacterMatcher.match() | 根据分类得分匹配角色 |
| 历史记录 | Storage.get().history | 按时间倒序排列 |

### 6.2 数据不变式

1. **题库数量**：仅显示真实题目数，不虚构
2. **分类统计规则**：
   - 0题分类：显示"未覆盖"，无进度条
   - 1-2题分类：显示"样本不足 · x/y"，无百分比，不参与强弱判断
   - ≥3题分类：显示标准化准确率百分比和x/y，参与强弱判断
3. **段位归一化**：保持现有段位系统不变
4. **知识/竞技分离**：知识掌握度和竞技表现独立计算和展示
5. **成就数据**：Storage.clearHistory()只清除history/highScore/last/lastTags，不影响achievements
6. **事件ID不变**：所有现有事件ID和触发逻辑保持不变

### 6.3 状态条数据限制

状态条仅显示以下真实现有数据：
- 题库数量（CONFIG.questions.length）
- 当前模式（easy/normal/hard）
- 成就数量（已解锁数/总数）
- 历史记录数量
- 当前分数
- 剩余时间

**禁止**：新增虚拟货币、养成系统、体力值等不存在的数据。

## 8. 无障碍规范

### 8.1 键盘导航

- 所有可交互元素支持Tab键焦点遍历
- 焦点顺序符合视觉阅读顺序
- Enter/Space键触发按钮点击
- Escape键关闭弹窗

### 8.2 Focus-Visible

- 所有可交互元素实现`:focus-visible`样式
- 焦点样式：3px品牌蓝(#315EFB)实线边框，3px偏移
- 按钮、卡片、徽章、开关均需实现

### 8.3 屏幕阅读器支持

- 所有图标按钮添加aria-label
- 状态变化添加aria-live区域
- 弹窗添加role="dialog"和aria-modal="true"
- 进度条添加aria-valuenow/aria-valuemin/aria-valuemax

### 8.4 对比度要求

- 文本与背景对比度≥4.5:1
- 按钮文字与背景对比度≥4.5:1
- 图标对比度≥3:1

### 8.5 减少动画支持

- 所有动画使用`prefers-reduced-motion`媒体查询
- 当用户设置减少动画时：
  - 禁用樱花花瓣飘落
  - 禁用卡片浮动动画
  - 禁用按钮悬停缩放
  - 保留必要的状态切换动画

## 9. 动画规范

### 9.1 动画类型

| 动画名称 | 触发场景 | 时长 | 缓动函数 | 适配prefers-reduced-motion |
|---------|---------|------|---------|---------------------------|
| 页面切换 | 页面进入 | 0.5s | cubic-bezier(0.34, 1.56, 0.64, 1) | 禁用 |
| 卡片进入 | 题目卡/模式卡 | 0.4s | cubic-bezier(0.34, 1.56, 0.64, 1) | 禁用 |
| 按钮按压 | 按钮点击 | 0.15s | cubic-bezier(0.4, 0, 0.2, 1) | 保留 |
| 按钮浮起 | 按钮悬停 | 0.25s | cubic-bezier(0.34, 1.56, 0.64, 1) | 禁用 |
| 正确反馈 | 答对题目 | 0.6s | cubic-bezier(0.34, 1.56, 0.64, 1) | 保留（简化） |
| 错误抖动 | 答错题目 | 0.5s | ease | 保留（简化） |
| 成就解锁 | 解锁成就 | 0.8s | cubic-bezier(0.34, 1.56, 0.64, 1) | 禁用 |
| 樱花飘落 | 背景动画 | 15-20s | linear | 禁用 |
| Combo弹跳 | 连击增加 | 0.5s | cubic-bezier(0.34, 1.56, 0.64, 1) | 保留（简化） |

### 9.2 动画实现原则

1. **轻微按压**：按钮点击时scale(0.96)，快速恢复
2. **浮起效果**：悬停时translateY(-3px) + 阴影加深
3. **高亮反馈**：选中时边框变色 + 发光阴影
4. **状态切换**：正确/错误答案的颜色过渡和图标显现
5. **性能优化**：使用CSS transform和opacity，避免重排
