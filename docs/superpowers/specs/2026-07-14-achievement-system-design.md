# 二次元浓度测试 · 成就系统设计文档

## 1. 概述

### 1.1 项目背景
本项目是「二次元浓度测试」——一款测试用户ACGN知识水平的趣味Web应用。为提升用户粘性和比赛展示效果，需要设计并实现一套成就系统。

### 1.2 设计目标
- **游戏化体验**：通过成就系统增加用户的挑战欲望和成就感
- **视觉冲击力**：参考皇室战争卡牌配色，打造炫酷的成就卡片效果
- **数据持久化**：成就解锁状态和时间持久化存储到LocalStorage
- **双位置展示**：结果页展示本次解锁成就，首页提供完整成就榜入口

### 1.3 评分标准适配
- **创新性**：独特的游戏化成就系统
- **实用性**：长期激励用户反复测试
- **完成度**：完整的成就检查、解锁、展示流程
- **设计**：高品质的视觉效果和动画

| achievements.progress.ranks | Array | 已获得的段位列表（去重） |
| achievements.progress.categoryStreak | Object | 各分类连续100%正确率次数 |

## 5. 逻辑流程

### 5.1 成就检查流程

```
测试完成 → 保存结果 → 检查成就条件 → 解锁新成就 → 更新存储 → 展示成就
```

### 5.2 核心逻辑

**测试完成时检查成就：**
1. 获取当前测试结果（分数、正确率、用时等）
2. 更新进度数据（连续次数、段位记录、分类进度）
3. 遍历所有成就，检查条件是否满足
4. 对满足条件且未解锁的成就，执行解锁操作
5. 保存解锁记录到LocalStorage

**解锁操作：**
1. 添加成就ID和解锁时间到unlocked数组
2. 触发解锁动画事件
3. 在结果页展示新解锁成就

## 8. 风险评估

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 成就条件过于复杂 | 逻辑出错概率增加 | 每个成就条件独立测试 |
| 动画效果过多影响性能 | 低端设备卡顿 | 使用CSS动画替代JS动画 |
| 数据结构变更影响旧数据 | 兼容性问题 | 实现数据迁移逻辑 |
| LocalStorage容量限制 | 存储失败 | 清理旧数据，保持成就数据精简 |
