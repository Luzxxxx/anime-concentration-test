# GitHub 作品集发布实施计划

## 目标

将现有单文件原生 Web 应用升级为可维护、可测试、可自动部署的公开 GitHub 作品集，同时保留核心玩法和樱花街机学院品牌。

## 阶段 1：基线与仓库卫生

1. 记录当前文件结构、关键页面和功能基线。
2. 用本地 HTTP 服务和真实浏览器跑通首页、模式选择、答题、结果、历史、成就与分享入口。
3. 创建 `.gitignore`，排除 `node_modules/`、`dist/`、`.superpowers/`、浏览器测试产物和 `demo.zip`。
4. 扫描仓库中的密钥、绝对路径和不适合公开的临时文件。

## 阶段 2：Vite 工程化与结构拆分

1. 创建 `package.json`、Vite 配置和 npm 脚本。
2. 将内联 CSS 从 `index.html` 提取到 `src/styles/`。
3. 将内联 JavaScript 提取到 `src/`，按数据、纯逻辑和 UI 职责拆分。
4. 将运行时静态资源移动到 `public/assets/` 并修正引用。
5. 保持现有 DOM 标识和用户流程，先确保行为等价，再做精修。

## 阶段 3：核心逻辑与自动测试

1. 将计分、抽题、题库校验和角色匹配整理为可独立导入的模块。
2. 使用 Vitest 覆盖正常路径、错误路径和边界条件。
3. 加入 ESLint 和格式检查，确保浏览器全局和模块边界明确。
4. 把题库校验接入测试和生产构建前检查。

## 阶段 4：UI 与体验精修

1. 精简首页信息层级，突出主行动按钮和项目价值。
2. 统一模式卡、答题卡、结果数据块和弹窗的设计令牌与交互状态。
3. 改善 375 px 手机端布局、触控目标、文字换行和安全区适配。
4. 优化加载失败、空历史、未解锁成就、分享不可用等状态。
5. 保留轻量动效并完整支持减少动效偏好。

## 阶段 5：PWA、元数据与版权边界

1. 修正 manifest、图标、主题色、应用名称和 Service Worker 资源清单。
2. 仅在生产环境注册 Service Worker，并确保相对路径适用于 GitHub Pages 子路径。
3. 添加 `LICENSE` 与 `THIRD_PARTY_ASSETS.md`，明确 MIT 不覆盖第三方图片和商标。
4. 保留并整理 `SOURCES.md`，修正明显的作品归属或描述错误。

## 阶段 6：GitHub 展示包装

1. 重写 README：首页封面、在线演示、亮点、截图、架构、运行、测试、部署和版权。
2. 添加贡献指南、问题模板和仓库元数据建议。
3. 创建 CI 工作流，在推送和拉取请求中执行 lint、test、build。
4. 创建 Pages 工作流，在 `main` 分支通过后部署 `dist/`。

## 阶段 7：验证与发布

1. 执行 `npm ci`、`npm run lint`、`npm run test` 和 `npm run build`。
2. 对构建产物执行真实浏览器桌面端与移动端冒烟测试。
3. 更新最终截图，确认 README 中的相对链接可在 GitHub 正确渲染。
4. 检查 Git 状态和提交内容，确保没有依赖、构建产物、临时文件或密钥。
5. 创建公开 GitHub 仓库、推送 `main`、启用 Pages，并验证线上地址。

## 完成定义

- 所有自动化命令通过。
- 核心功能在开发环境和生产构建中均可用。
- 公开仓库内容、许可证和第三方素材边界清晰。
- GitHub Pages 在线地址可访问，README 可直接引导访客试玩和复现项目。
